# 辞書エディションの収録区分

## 決定

正規見出し語の各検索結果には、そのエントリを収録する最小の公式SudachiDictエディションを表示します。

- `収録: Small`は、Small・Core・Fullのすべてに収録されていることを表します。
- `収録: Core`は、Smallにはなく、CoreとFullに収録されていることを表します。
- `収録: Full`は、Fullだけに収録されていることを表します。

サンプルfixtureは公式SudachiDictエディションではないため、収録区分を表示しません。

## コンパクトな表現

固定した`20260428`リリースの公式エディションは、word IDの累積範囲になっています。完全な中間形式の書き出しから、次の境界を確認しています。

| 最小エディション | 正規word IDの範囲 | ソースレコード数 |
| --- | ---: | ---: |
| Small | `0`–`765648` | 765,649 |
| Core | `765649`–`1629079` | 863,431件の追加 |
| Full | `1629080`–`2883176` | 1,254,097件の追加 |

マニフェストの`editionMembership`には、`smallUpperExclusive`と`coreUpperExclusive`だけを保存します。Workerは正規見出し語IDをこの境界と比較し、画面へ`Small`・`Core`・`Full`のいずれかを返します。ブラウザ用レコードshardにエディション用のbyteや重複文字列を追加する必要はありません。

収録区分は表層文字列ではなく辞書レコードに属します。そのため、同じ表記を持つ別レコードに異なる区分を表示できます。検索結果はすでに辞書形の正規word IDへ解決されるため、活用形レコードによって画面上の例外が生じることもありません。

## リリース時の検証

word IDの順序は、将来のすべてのSudachiDictに対する仮定ではなく、固定したリリースごとに検証する性質です。本番workflowでは三つの公式パッケージをすべてインストールして書き出し、次の条件を一行ずつ検証します。

1. Smallが設定どおりのレコード数であること。
2. Smallの全レコードが、CoreおよびFullの対応レコードと同一であること。
3. Coreが設定どおりのレコード数であること。
4. Coreの全レコードが、Fullの対応レコードと同一であること。
5. Fullが設定どおりのレコード数であること。

境界が空、順序不正、または対象データセットの外にある場合、生成を失敗させます。ブラウザデータの検証でも、マニフェストの境界値とエントリ数が`config/dictionary-release.json`と一致することを要求します。将来のリリースでエディションが累積関係でなくなった場合は、この範囲規則を暗黙に使い続けず、明示的な収録区分表現へ変更します。
